# reliability

Purpose: Restore service first, identify the real failure mode, then harden the path against recurrence.

Capabilities:
- triage incidents and runtime failures
- contain performance, memory, stream, and lifecycle regressions
- audit async, thread-safety, locking, and event ordering
- inspect logs, anomaly signatures, and uptime evidence
- enforce resource budgets and launch-window gates
- verify fallback, recovery, and cleanup behavior
- produce stabilization plans and recurrence guards
