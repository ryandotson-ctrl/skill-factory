# Planning Contracts

Use these contracts to keep planning outputs concrete and execution-ready.

## BacklogItemV1
- `title`
- `owner_lane`: backend, frontend, qa, ops, product, or mixed
- `definition_of_done`
- `dependencies`
- `priority`
- `size`
- `risks`

## MilestoneViewV1
- `now`
- `next`
- `later`
- `critical_path`
- `blocked_by`

## RiskRegisterEntryV1
- `risk`
- `impact`
- `likelihood`
- `mitigation`
- `trigger_to_revisit`

## StatusUpdateV1
- `done`
- `in_progress`
- `blocked`
- `next`
