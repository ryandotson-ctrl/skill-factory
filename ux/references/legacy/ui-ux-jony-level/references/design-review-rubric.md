# Design Review Rubric

Score each category from 0 to 5, then list the top 5 fixes in priority order.

## Score Categories
- hierarchy and legibility (25)
- information architecture and discoverability (20)
- interaction clarity and error prevention (20)
- accessibility and inclusivity (15)
- consistency and token discipline (10)
- performance and perceived speed (10)

## What Good Looks Like
- hierarchy is obvious in 3 seconds
- primary action is unambiguous
- states and recovery paths are fully specified
- spacing and typography are systemized
- motion clarifies causality
- implementation risk is visible, not hidden

## Top 5 Fix Rules
- each fix should be concrete
- rank by user impact first
- call out regression risk if a fix touches hierarchy, spacing, states, or trust cues
