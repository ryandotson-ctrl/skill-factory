# Style Modes

The default bar remains Apple-grade craft, but not every product should look the same.

## `apple-default`
Use when the product needs calm precision, reduced surfaces, strong hierarchy, disciplined motion, and premium restraint.

Traits:
- minimal surfaces
- generous whitespace
- restrained depth
- high material honesty

## `editorial-premium`
Use when narrative, storytelling, or product marketing clarity needs more dramatic typography and contrast than the default mode.

Traits:
- stronger typographic voice
- bolder section contrast
- more expressive rhythm
- still disciplined, not ornamental

## `operator-utility`
Use for dashboards, control surfaces, runtime tools, and dense workflows where trust, status, and scan speed outrank aesthetic softness.

Traits:
- compact but legible density
- explicit status hierarchy
- reduced decorative depth
- stronger error and action affordances

## `warm-human`
Use when onboarding, coaching, or emotionally sensitive flows need reassurance and warmth without becoming cute or cluttered.

Traits:
- softer contrast transitions
- supportive microcopy
- clearer empty and recovery states
- approachable motion and feedback

## Selection Rules
- default to `apple-default` unless the product need strongly indicates otherwise
- explain why a non-default mode was selected
- style mode changes must not weaken accessibility or hierarchy
