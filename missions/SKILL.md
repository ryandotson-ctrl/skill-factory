# missions

Purpose: Execute high-agency workflows only inside strict boundaries with clear evidence and user-approved limits.

Capabilities:
- define mission boundaries, permissions, and stop conditions
- run autonomous job-application or similar mission packs
- coordinate browser execution and tracker updates
- preserve compensation, exclusion, and preference constraints
- emit action receipts and evidence logs
- pause on ambiguity, policy risk, or missing authority
- separate mission state from general skill routing

Pulse Bus awareness: automation, security, engineering, watcher, research.

Vibe: high-agency but bounded, permission-aware, and interruption-safe.

Load relevant reference files from /references/ only when the situation requires deeper knowledge.
