# Platform Translation

Use this when adapting the design between web, iOS, and visionOS.

## Web To iOS
- map token intent, not exact pixel values
- convert web actions to native control patterns where platform conventions are stronger
- keep hierarchy and content rhythm intact
- preserve accessibility rules for focus, contrast, and reduced motion

## Web To visionOS
- prioritize comfort and depth discipline
- use focus-driven navigation rather than desktop-style density
- keep depth meaningful, not decorative
- avoid visual overload in layered space

## Translation Checks
- preserve the primary task path
- preserve trust cues and status clarity
- preserve error recovery and confirmation semantics
- explain any platform-specific interaction changes explicitly
